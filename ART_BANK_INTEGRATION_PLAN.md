#### 4. Система Управления Правами Доступа ⭐⭐⭐⭐⭐
**Для реализации связи Консультант ↔ Коллекционер**

**Требуется:**
- Таблица разрешений (permissions)
- UI для Коллекционера: раздел «Консультанты» с кнопками выдачи/отзыва прав
- UI для Консультанта: запрос доступа к клиенту

**Типы прав:**
1. `VIEW_PORTFOLIO` - Просмотр портфеля
2. `VIEW_WISHLIST` - Просмотр статистики желаний
3. `MAKE_DEALS_WITH_CONFIRM` - Совершение сделок с подтверждением
4. `FULL_MANAGEMENT` - Полное управление

**Новая таблица:**
```sql
CREATE TABLE consultantPermissions (
  id INTEGER PRIMARY KEY,
  consultantId INTEGER,
  collectorId INTEGER,
  permissions TEXT, -- JSON массив ['VIEW_PORTFOLIO', 'MAKE_DEALS_WITH_CONFIRM']
  status TEXT, -- 'pending', 'approved', 'rejected'
  requestDate TEXT,
  approvedDate TEXT
);
```

**API Endpoints:**
```typescript
consultantPermissions: {
  request: protectedProcedure.mutation(), // Консультант запрашивает
  approve: protectedProcedure.mutation(), // Коллекционер одобряет
  revoke: protectedProcedure.mutation(),  // Коллекционер отзывает
  getByCollector: protectedProcedure.query(),
  getByConsultant: protectedProcedure.query(),
}
```
